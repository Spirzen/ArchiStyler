# ArchiStyler

**ArchiStyler** — настольное приложение для визуального проектирования архитектуры на **C#** и **Java**.  
Вы рисуете UML-подобную диаграмму классов, задаёте роли и связи, применяете шаблоны паттернов — и получаете готовые исходники с файлами проекта.

## Зачем это нужно

ArchiStyler сокращает путь от идеи архитектуры до рабочего каркаса кода:

| Этап | Без ArchiStyler | С ArchiStyler |
|------|-----------------|---------------|
| Набросок слоёв | Блок-схема в draw.io / на бумаге | Диаграмма с живыми классами |
| Каркас классов | Ручное создание файлов | Шаблоны MVP, Repository, GoF… |
| Связи и зависимости | Комментарии в коде | Стрелки наследования, implements, uses |
| Экспорт в IDE | Копипаст | `.cs` / `.java` + `.csproj` / `pom.xml` |

Приложение ориентировано на прототипирование, обучение паттернам и быструю стандартизацию слоёв в небольших командах.

## Ключевые возможности

| Область | Что умеет |
|---------|-----------|
| **Диаграмма** | Drag-and-drop карточек классов, папки на холсте, масштаб и панорама |
| **Связи** | Ручное соединение точек на рёбрах карточек; автоматические стрелки из инспектора |

| **UI** | Адаптивная вёрстка, тёмная неоновая и светлая темы, встроенная справка |

## Быстрый старт

### Способ 1 — `run.bat` (рекомендуется)

Двойной клик по файлу **`run.bat`** в корне проекта.  
Окно консоли останется открытым после выхода; при сбое будет показан `crash.log`.

### Способ 2 — командная строка

```powershell
cd "F:\Projects\C#\ArchiStyler"
dotnet restore
dotnet run --project src\ArchiStyler\ArchiStyler.csproj
```

### Способ 3 — Visual Studio / Rider

1. Откройте **`ArchiStyler.slnx`**
2. Установите **`ArchiStyler`** стартовым проектом
3. Нажмите **F5**

### Первый проект за 2 минуты

1. На **стартовом экране** выберите язык (C# или Java), имя проекта, папку экспорта и namespace/package по умолчанию.
2. В главном окне нажмите **«Применить»** у шаблона (например, MVP или Repository) **или** **«+ Класс»**.
3. Кликните класс на диаграмме — справа отредактируйте имя, члены, модификаторы.
4. Соедините классы: потяните с **цветной точки** на краю карточки к точке другого класса.
5. Вкладка **«Код»** — превью; кнопка **«Экспорт + проект»** — файлы на диск.

## Интерфейс и рабочий процесс

Главное окно разделено на зоны:

```
┌─────────────────────────────────────────────────────────────────┐
│  ArchiStyler · Имя · Язык · Путь    [Справка][Тема][Сохр][Эксп] │
├──────────┬──────────────────────────────────────┬───────────────┤
│  Левая   │                                      │   Инспектор   │
│  панель  │         Холст диаграммы              │   класса /    │
│          │    (классы, папки, стрелки)          │   вкладка     │
│  Классы  │                                      │   «Код»       │
│  Паттерны│                                      │               │
└──────────┴──────────────────────────────────────┴───────────────┘
│                        Строка статуса                           │
└─────────────────────────────────────────────────────────────────┘
```

### Левая панель

- **+ Класс** — пустой класс на холсте
- **+ Папка** — контейнер (Models, Services…) для группировки и путей экспорта
- **Очистить всё** — сброс диаграммы и превью (файлы на диске не трогаются)
- **Тип новой связи** — значение по умолчанию при ручном соединении
- **Роль** + **Добавить по роли** — класс с типовыми членами
- **Шаблон** + **Применить** — вставка готового паттерна

### Инспектор (правая панель)

Редактирование выбранного класса:

- имя, namespace/package, XML/summary-комментарий
- флаги: `interface`, `abstract`, `enum`, `sealed`
- базовый тип и реализуемые интерфейсы
- список `using` / `import`
- члены: поля, свойства, методы, конструкторы
- опция **«заглушка»** — тело метода с `TODO` или `return` по умолчанию

### Вкладка «Код»

- **Обновить из модели** — сгенерировать текст из диаграммы
- **Применить код → модель** — обратный разбор редактора в модель класса

## Диаграмма и связи

### Навигация по холсту

| Действие | Управление |
|----------|------------|
| Масштаб | **Ctrl + колёсико** или кнопки **+ / − / Сброс** |
| Панорама | **Средняя кнопка мыши** + перетаскивание |
| Панорама (альт.) | **Пробел** + левая кнопка по пустому месту |
| Перемещение класса | Перетащить карточку левой кнопкой |
| Выбор | Клик по карточке |

### Папки на диаграмме

Папки (📁) группируют классы визуально и влияют на **путь экспорта**:

- перетащите класс **в папку** или **на корень** — обновятся namespace/package и относительный путь файла
- сегмент папки (`Segment`) добавляется к базовому namespace проекта

Пример для C#: `DefaultNamespace = App` + папка `Services` → `App.Services.UserService`.

### Типы связей

| Тип | Смысл | Пример в коде |
|-----|--------|---------------|
| **Inherits** | Наследование | `class B : A` |
| **Implements** | Реализация интерфейса | `class S : IService` |
| **Uses** | Зависимость / использует | поле, параметр |
| **Composes** | Сильная композиция | владеет жизненным циклом |
| **Aggregates** | Агрегация | слабая связь «часть–целое» |
| **FieldReference** | Ссылка на поле | существующее или новое |
| **MethodReference** | Ссылка на метод | вызов / зависимость |
| **UsingImport** | Импорт пространства имён | `using` / `import` |

**Как создать связь вручную:** потяните с цветной точки на краю карточки к точке другого класса → откроется диалог выбора типа.

**Автоматические связи:** если в инспекторе указать **базовый тип** или **интерфейсы**, стрелки появятся на диаграмме без ручного соединения.

**Редактирование:** перетащите конец стрелки к другой точке; удалите через **Delete** или **ПКМ**.

## Роли классов

Роль задаёт семантику класса в архитектуре и добавляет **типовые члены-заготовки**.

Примеры ролей:

| Категория | Роли |
|-----------|------|
| Данные | DTO, Entity, Model, Mapper |
| Доступ к данным | DAO, Repository, OrmContext, DatabaseClient |
| Бизнес-логика | Service, Handler, Strategy, Command |
| Инфраструктура | Logger, Config, HttpClient, ErrorHandler |
| UI / архитектура | View, Presenter, ViewModel, Controller |
| Прочее | Factory, Builder, Adapter, Validator, EntryPoint… |

Полный перечень — в enum `ClassRole` (`src/ArchiStyler/Models/ClassRole.cs`), отображается в ComboBox с человекочитаемыми подписями.

## Шаблоны паттернов

Шаблоны хранятся в JSON-файлах:

```
src/ArchiStyler/Assets/Templates/
├── patterns.json       # MVP, MVVM, Singleton, Strategy, Layered, Repository+ORM…
└── patterns-gof.json   # GoF: Abstract Factory, Observer, Decorator, Visitor…
```

Все файлы `Assets/Templates/*.json` **подхватываются автоматически** при запуске — можно добавлять свои без изменения кода.

### Встроенные шаблоны

**patterns.json**

| ID | Название | Категория |
|----|----------|-----------|
| mvp | MVP | Architectural |
| mvvm | MVVM | Architectural |
| singleton | Singleton | Creational |
| factory | Factory Method | Creational |
| strategy | Strategy | Behavioral |
| command | Command | Behavioral |
| repository-orm | Repository + ORM | Data |
| adapter | Adapter | Structural |
| layered | Layered Architecture | Architectural |

**patterns-gof.json**

Abstract Factory, Prototype, Bridge, Composite, Decorator, Facade, Flyweight, Proxy, Observer, State, Template Method, Chain of Responsibility, Mediator, Memento, Visitor, Builder.

### Формат JSON-шаблона

```json
{
  "patterns": [
    {
      "id": "my-pattern",
      "name": "My Pattern",
      "category": "Custom",
      "description": "Краткое описание",
      "folders": [
        { "name": "Services", "segment": "Services", "offsetX": 0, "offsetY": 0, "width": 320, "height": 240 }
      ],
      "classes": [
        {
          "name": "MyService",
          "role": "Service",
          "folder": "Services",
          "offsetX": 20,
          "offsetY": 40,
          "isInterface": false,
          "baseType": "",
          "implements": [],
          "usings": [],
          "members": [
            { "kind": "Method", "name": "Run", "returnType": "void", "access": "Public", "generateStub": true }
          ]
        }
      ],
      "relations": [
        { "from": "MyService", "to": "IMyService", "kind": "Implements" }
      ]
    }
  ]
}
```

Допустимые значения `kind` для связей: `Inherits`, `Implements`, `Uses`, `Aggregates`, `Composes`.  
Для членов: `Field`, `Property`, `Method`, `Constructor`.

## Генерация и обратный разбор кода

### Генерация (модель → код)

- `CSharpCodeGenerator` — классы C# с namespace, usings, модификаторами, свойствами и заглушками методов
- `JavaCodeGenerator` — аналог для Java с package и import

Генератор выбирается по языку проекта через `CodeGeneratorFactory`.

### Обратный разбор (код → модель)

Парсеры в `Services/Parsing/`:

- `CSharpCodeParser` — эвристический разбор C# (regex, без Roslyn)
- `JavaCodeParser` — аналог для Java

**Что распознаётся:** объявление типа, namespace/package, usings, базовый тип, интерфейсы, поля, свойства, методы, конструкторы.

**Ограничение:** парсер упрощённый. Для сложного кода (generics, partial classes, records с нестандартным синтаксисом) **всегда проверяйте результат** на вкладке «Код» и диаграмме.

## Экспорт и сохранение

### Экспорт исходников

Кнопка **«Экспорт + проект»** создаёт:

| Язык | Файлы |
|------|-------|
| **C#** | `.cs` по структуре namespace + `{ИмяПроекта}.csproj` |
| **Java** | `.java` по структуре package + `pom.xml` |

- каталоги создаются автоматически
- существующие файлы **перезаписываются**
- ошибки по отдельным классам собираются в строку статуса

### Сохранение диаграммы

Кнопка **«Сохранить»** записывает `{ИмяПроекта}.archistyler.json` в папку проекта (`RootPath`).

Файл содержит полную модель: классы, связи, папки, координаты, настройки.  
Загрузка сохранённых проектов через UI — в развитии; формат JSON стабилен для ручного использования и будущих версий.

## Расширение и кастомизация

### Добавить свой паттерн

1. Создайте `src/ArchiStyler/Assets/Templates/patterns-custom.json`
2. Скопируйте структуру из существующих файлов
3. Пересоберите проект — шаблон появится в ComboBox

### Добавить роль

1. Расширьте enum `ClassRole`
2. При необходимости добавьте подпись в `RoleCatalog` и заготовки в `TemplateService.ApplyRoleScaffold`

### Изменить тему

Редактируйте ресурсы в `Assets/Themes/AppThemeResources.axaml`.

## Ограничения и известные нюансы

- **Парсер кода** — эвристический, не заменяет Roslyn/JavaParser; для production-кода используйте как отправную точку.
- **Загрузка `.archistyler.json`** через UI — запланирована; сохранение уже работает.
- **Экспорт перезаписывает** файлы в `RootPath` без предупреждения.
- **Очистить всё** удаляет только модель в памяти, не файлы на диске.
- Приложение ориентировано на **Windows Desktop** (Avalonia Desktop).

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| `dotnet` не найден | Установите [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0) |
| Ошибка при запуске через `run.bat` | Смотрите код выхода и `src/ArchiStyler/bin/Debug/net8.0/crash.log` |
| Шаблоны не видны | Проверьте JSON в `Assets/Templates/`; невалидные файлы пропускаются |
| Экспорт не создаёт файлы | Убедитесь, что `RootPath` существует и доступен на запись |
| Код после «Применить» не совпадает | Упростите исходник или правьте модель вручную в инспекторе |

Встроенная справка: кнопка **«Справка»** в главном окне.
